# R2C エラーメッセージ規則（パートナーフレンドリー）

R2Cのパートナーは心理学×営業の専門家であり、IT専門家ではない。エラー画面で技術用語を見せると不安・困惑させてしまう。

## 絶対NG（画面に出してはいけない言葉）

- HTTPステータス: `401` `403` `404` `500` `Unauthorized` `Forbidden` `Not Found`
- 認証技術: `JWT` `Token` `Bearer` `OAuth` `Session expired`
- ネットワーク: `CORS` `CSP` `Fetch failed` `Network Error` `Timeout`
- DB/サーバー: `SQL` `Query failed` `Database error` `Internal Server Error`
- フロント技術: `Promise` `undefined` `null` `Cannot read property` `TypeError`
- 英語のままのエラーメッセージ

## 必須ルール

1. **専門用語を「やったこと」に翻訳する**
   - 「Unauthorized」→「ログイン状態が切れたみたいです」
   - 「Validation error」→「入力内容に少しだけ問題があります」
   - 「Network error」→「サーバーとうまくお話できませんでした」

2. **解決策を必ず添える**
   - ❌「エラーが発生しました」
   - ✅「読み込みに失敗しました。少し時間をおいてもう一度試してみてください 🙏」

3. **責任を「あなたのせい」にしない**
   - ❌「正しい形式で入力してください」（責められてる感）
   - ✅「メールアドレスの形式が違うかもしれません。@マークが入っているか確認してね」

4. **絵文字は控えめに（1メッセージに最大1個）**
   - 🙏 (お願い系) / 😊 (成功系) / 💡 (ヒント系) / ⚠️ (注意系、ただし多用しない)
   - NG: 🚫🛑❌💥 等の強い否定絵文字

5. **空状態は「これからどうするか」を促す**
   - ❌「データがありません」（白い画面感）
   - ✅「まだ書籍が登録されていません。右上の『＋追加』ボタンから最初の1冊をアップロードしてみてください 📚」

## 標準テンプレート集

### 認証系

```typescript
// 401 (Unauthorized)
"ログイン状態が切れたみたいです。お手数ですがもう一度ログインしてください 🙏"

// 403 (Forbidden / 他テナントへのアクセス)
"この情報は閲覧できないようです。担当の方にご確認ください"

// セッションタイムアウト
"しばらく操作がなかったので、もう一度ログインが必要です"
```

### バリデーション系

```typescript
// 必須項目
"○○ は入力が必要です"

// メール形式
"メールアドレスの形式が違うかもしれません。@マークが入っているか確認してね"

// 文字数オーバー
"○○ は●●文字までで入力してください（現在 ▲▲文字）"

// PDF以外のファイル
"アップロードできるのはPDFファイルだけです。ファイルの拡張子が .pdf になっているか確認してね 📄"

// ファイルサイズ超過
"ファイルが少し大きすぎるみたいです（●●MBまで）。圧縮するか、分割してみてください"
```

### API/ネットワーク系

```typescript
// 5xx / 不明エラー
"うまく読み込めませんでした。少し時間をおいてもう一度試してみてください 🙏"

// タイムアウト
"サーバーの応答に時間がかかっています。もう一度試してみてください"

// オフライン
"インターネットに接続できないみたいです。Wi-Fiや電波の状態を確認してね"

// レート制限
"アクセスが集中しているようです。1〜2分ほど待ってからもう一度試してみてください"
```

### 空状態（Empty State）

```typescript
// 書籍未登録
"まだ書籍が登録されていません。右上の『＋追加』ボタンから最初の1冊をアップロードしてみてください 📚"

// 会話履歴ゼロ
"まだお客様との会話がありません。ウィジェットがサイトに正しく埋め込まれているか確認してね"

// 検索結果ゼロ
"見つかりませんでした。別のキーワードで試してみてね 💡"

// チューニングルールゼロ
"まだチューニングルールが設定されていません。AIにこう答えてほしい、というルールを追加してみましょう"
```

### ローディング状態

```typescript
// 短時間
"読み込み中..."

// 中時間（5〜15秒予想）
"少々お待ちください..."

// 長時間（PDF処理等）
"書籍を分析しています。数十秒かかることがあります 📖"
```

### 成功メッセージ

```typescript
// 保存成功
"保存しました ✓"

// アップロード成功
"アップロードできました 📚 ナレッジに反映されるまで少し時間がかかります"

// 削除成功
"削除しました"
```

## 実装パターン（Admin UI / React + Shadcn）

```tsx
// toast 例（既存パターン: admin-ui/src/lib/toast.ts に集約）
import { toast } from "@/lib/toast";

try {
  await api.uploadBook(file);
  toast.success("アップロードできました 📚 ナレッジに反映されるまで少し時間がかかります");
} catch (err) {
  // ❌ NG: toast.error(`Upload failed: ${err.message}`);
  // ✅ OK:
  const message = err.status === 413
    ? "ファイルが少し大きすぎるみたいです（50MBまで）。圧縮するか、分割してみてください"
    : "アップロードに失敗しました。少し時間をおいてもう一度試してみてください 🙏";
  toast.error(message);

  // 技術詳細はconsole.errorに残す（開発者向け）
  console.error("[uploadBook]", err);
}
```

## Widget側のエラー（visitor向け）

エンドユーザー（visitor）向けはさらに優しく。商談中に表示される可能性があるため。

```typescript
// チャットAPI失敗
"うまくお返事できませんでした。もう一度お試しいただけますか？"

// 接続エラー（visitor側）
"通信が安定しないようです。少し時間をおいてもう一度話しかけてみてください"
```
