# Asana タスク記述規約

> 最終更新: 2026-05-19（Phase70-J）
> 目的: 24h 自走中の Asana Watcher（Phase70-D）が「自走可能タスク」を機械的にフィルタできるよう、タスク記述を統一する。

## タイトル規約

### 接頭辞（必須）

| 接頭辞 | 用途 |
|--------|------|
| `feat:` | 新機能実装 |
| `docs:` | ドキュメントのみ変更 |
| `fix:` | バグ修正 |
| `schema:` | DB スキーマ変更 |
| `chore:` | ビルド・設定・依存更新 |
| `refactor:` | 振る舞いを変えないリファクタリング |

### 禁止文字

タイトル先頭に以下の文字を使わないこと（Asana MCP で作成失敗の原因）:

| 文字 | NG 例 |
|------|-------|
| `[` | `[Tier S] feat: ...` |
| `~` | `~Phase70-J~` |
| `.` | `.env 更新` |

### 命名例

```
✅ feat: Phase70-D タスクキュー（Asana Watcher）実装
✅ docs: ASANA_TASK_TEMPLATE.md 作成
✅ fix: ES index naming 不整合修正（Phase33-c 起因）
❌ [Tier S] 安全装置構築      ← [ は禁止
❌ Phase70-A.                  ← 末尾 . は禁止
```

## description 必須構造

```
Tier: S | A | B
Parent: Phase番号 (GID: XXXXXXXXXX)   ← 親タスクがある場合のみ記載

## 目的
1〜3文で「なぜやるのか」を説明する。

## 背景
（任意）問題発見の経緯・技術的負債の詳細・依存タスクとの関係。

## DoD
- [ ] 完了条件1（測定可能な形で記述）
- [ ] 完了条件2

## 推奨モデル
Opus 4.7 / Sonnet 4.6 / Plan Mode（理由を1文で）

## 関連
- タスク名 (GID): 依存関係の種類
- docs/XXXX.md: 参照ドキュメント

## 一切しないこと
- やってはいけない操作を箇条書き

/goal このタスクで達成したい状態を1文で。Asana Watcher がタスクの意図を読み取るための要約。
```

## メタ情報規約

### Tier（3段階）

| Tier | 対象 | 24h 自走 |
|------|------|----------|
| **S** | 本番サービス影響・セキュリティ・VPS 操作・avatar-agent | 原則 NG（人間専権） |
| **A** | 機能追加・複数ファイル横断・API 変更 | 条件付き可（`24h-eligible` タグ付与で有効化） |
| **B** | docs・設定・スクリプト・テストのみ | 自走可（タグ不要、自動フィルタ対象） |

Tier S / A / B 以外の文字列は使用禁止。

### 推奨モデル（3択）

| 値 | 用途 |
|----|------|
| `Opus 4.7` | セキュリティ変更・複雑なマルチファイル・本番データ移行計画 |
| `Sonnet 4.6` | 通常の機能追加・docs 作成・スクリプト・Asana MCP 操作 |
| `Plan Mode` | 設計検討・要件整理・戦略判断 |

## 24h-eligible タグ運用

### 概要

24h 自走中の Asana Watcher（Phase70-D GID: 1214919660548852）は以下の条件でタスクを補給対象とする:

1. **Tier B タスク** — 常に自走対象（タグ不要）
2. **Tier A + `24h-eligible` タグ** — 人間が明示的に「自走可」と判断したもの

Tier S は `24h-eligible` タグがあっても自走対象にならない。

### タグ情報

| 項目 | 値 |
|------|-----|
| タグ名 | `24h-eligible` |
| GID | `1214922984195645` |
| 色 | `light-green` |
| ワークスペース | `mooores.com` (817733952351708) |

### 付与ルール

- **付与者**: hkobayashi が Asana で手動付与
- **付与条件**: Tier A で、かつ以下を全て満たす
  - VPS 操作不要
  - DB マイグレーション不要
  - `avatar-agent/` への変更なし
  - 法務・パートナー契約ドキュメントに触れない
  - CODEOWNERS で保護されたファイルへの変更なし
- **禁止**: 自動付与スクリプトの作成（手動運用で開始）
- **禁止**: Tier S タスクへの付与

### Asana Watcher フィルタロジック（参考）

```bash
# Tier B → 常に補給対象
# Tier A + 24h-eligible タグ → 補給対象
# Tier S → 常に除外（タグがあっても無効）
# DB migration 必要タスク → 除外（人間専権）
```

## 遡及適用ルール

### 対象

- 進行中（未完了）Phase70 サブタスク: 70-B 〜 70-L（70-A・70-G は完了済のため対象外）
- 進行中 Phase69-2: B / D / E（69-2-A・完了済 Phase69 タスクは対象外）

### 方法

**description の補完のみ。タイトル変更は一切しない**（URL・Asana 通知が壊れる）。

補完の優先度:
1. `/goal` が未記載 → 追加
2. `Tier:` が未記載 → 追加
3. `## 推奨モデル` が未記載 → 追加
4. `Parent:` が未記載 → 追加（Phase70 系: GID 1214919472827777）

### 対象外

- 完了済タスク（Phase70-A, 70-G, Phase69-2-A 等）
- Phase69 完了タスク全般
