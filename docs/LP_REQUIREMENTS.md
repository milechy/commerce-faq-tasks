## 0. LP の核心思想

**「読む前に使わせる」**

LP を下スクロールするより先に、訪問者にアバターとの会話を体験させる。
読んで納得 → 試してみる、ではなく、試して体験 → 読んで確信、の順序に設計する。

## 4. Section 1: HERO 詳細

### コピー（痛み解決型・改善版）

```
キャッチ:
「あなたのサイトに、24時間365日・休まず働くAIセールス担当を。」

サブ:
「1行の埋め込みだけ。訪問者の『わからない』を3秒で解決し、
購入・予約・問い合わせまで自動誘導。
夜中の離脱を防ぎ、売上を逃さない。」

数値アンカー（Social Proof から引用):
「平均導入期間 3日 / 夜間対応コスト 0円 / 累計対話 XX万件」
```

### アバターデモ仕様

| 項目 | 仕様 |
|---|---|
| 使用テナント | `r2c_default`（Phase66 で作成済み） |
| デスクトップ表示 | Hero 右半分にインライン埋め込み（PC グリッド 3fr:2fr / 幅 900px）。スクロールで Hero が画面外に出たら右下フローティングに自動切替 |
| モバイル表示 | **自動展開なし**（即離脱リスクを回避）。フキダシ形式のツールチップ「こんにちは！R2C です 👋」のみ表示し、タップで全画面展開 |
| 初期グリーティング | 「こんにちは！R2C のデモ AI です。料金・機能・導入方法など、何でも聞いてください。」 |
| avatar fallback | LiveKit 未接続時は静止画（PNG）を表示 |

#### Hero ↔ Section 7 の役割分担（カニバリズム解消）

Gemini 指摘: Section 1 と Section 7 で「デモ体験」が被るとユーザーが迷う。以下の分担で解消する。

| セクション | 役割 | ウィジェットの状態 |
|---|---|---|
| **Section 1 (Hero)** | 「このサイトに AI がいること」を体感させる | インライン埋め込み、小サイズ表示 |
| **Section 7 (Live Demo)** | 腰を据えて使ってもらう + CTA へ誘導 | 大型展開 + 例示質問ボタン + 3 ターン後 CTA |

Hero のウィジェットは「存在を示す」、Section 7 は「深く体験させる」、と明確に分ける。

## 11. Section 11: FAQ（AI 特有の懸念を先回り・拡充版）

### 基本

- **導入まで本当に最短 3 日でできますか？**  
  → ヒアリング（1 日）→ テナント開通・1 行埋め込み（1 日）→ ドライラン確認（1 日）が標準フロー。IT 担当不要。

- **既存のサイトに影響はありますか？**  
  → Shadow DOM で隔離されているため、既存の CSS・JS には一切干渉しません。

### 精度・品質

- **AI の回答精度はどれくらいですか？**  
  → 登録した FAQ・商品情報のみを参照する RAG 方式で、でたらめな回答（ハルシネーション）を防ぎます。Gemini による 4 軸 Judge 評価でスコアを継続モニタリングします。

- **わからない質問への回答は？**  
  → 「確認してご連絡します」と伝え、人間スタッフへの引き渡しフローに切り替えます。

### プライバシー・セキュリティ

- **顧客の会話データは安全ですか？**  
  → 会話ログは AES-256-GCM で暗号化。日本国内サーバー（Hetzner）に保管。第三者への提供・学習への使用は行いません。

- **個人情報（PII）の扱いは？**  
  → PII 検出フィルターが自動で作動し、個人情報をログ・メトリクスに含めない設計です。

### 連携・運用

- **人間スタッフとの連携はどうなりますか？**  
  → AI が対応できない質問・高優先度の訪問者は、スタッフへの引き渡しトリガーを設定できます。

- **Intercom / LINE / 他のチャットと併用できますか？**  
  → 技術的には共存可能。ただし同一サイト上での複数チャットウィジェット表示は UX 上推奨しません。

- **AI が間違えた場合、後から修正できますか？**  
  → Admin UI から FAQ・回答内容をリアルタイムで更新できます。変更は即時反映されます。

## 12. Section 12: LEAD CAPTURE FORM

### フォーム仕様

```
フィールド:
  - お名前（必須）
  - 会社名（必須）
  - サイト URL（必須 / https://）
  - メールアドレス（必須）
  - 業種セレクタ（EC / 美容 / フィットネス / 医療 / 士業 / その他）
  - ご要望・質問（テキストエリア、任意）
  - honeypot フィールド（hidden / スパム防止）
  - Cloudflare Turnstile（ユーザーにパズルを解かせず、プライバシー配慮型 bot 防止）
  - [無料デモを申し込む] ボタン（min-height: 52px）

送信後:
  - サンクスページ / インラインメッセージ
    「3 営業日以内にご連絡します。
     その間、デモ Admin UI をご覧いただけます。→ [admin.r2c.biz]」
```

### バックエンド処理

```
POST /v1/public/lead-inquiry
  Body: { name, company, siteUrl, email, industry?, message? }
  Auth: なし（rateLimit: 10 req/hour per IP）
  Validation: Zod スキーマ（email format, https:// prefix）

  Action:
    1. leads テーブルに INSERT
    2. SendGrid: 申込者宛 自動返信メール
    3. SendGrid: R2C 担当宛 通知メール（hkobayashi@mooores.com）
    4. Asana API: 「新規リード: {company}（{industry}）」タスク作成
  
  rateLimit: 30〜50 req/hour per IP
    ※ Gemini 指摘: 10 req/hour では大企業の共有 IP（社内 proxy）から
      複数担当者が同時にフォームを試した際にブロックしてしまう。
      Cloudflare Turnstile で bot を防ぐため、IP 上限は緩和でよい。
  
  Response: { success: true }
  Error:    { success: false, message: "..." }（ユーザー向け friendly メッセージ）
```

## 13. 技術要件

### LP 本体

| 項目 | 選択 | 理由 |
|---|---|---|
| 実装 | 静的 HTML + Tailwind CSS（Vite ビルド） | admin-ui の設計資産を再利用、SSG 不要 |
| ホスティング | `r2c.biz`（Cloudflare Pages / Nginx 静的） | 既存 Cloudflare 構成と整合 |
| ウィジェット | `https://cdn.r2c.biz/widget.js` | Phase C の CDN 配信前提 |
| デモテナント | `r2c_default`（Phase66 作成済み） | 専用 LP デモ API キーを新規発行 |
| Lead API | 新規エンドポイント `POST /v1/public/lead-inquiry` | 認証不要・rateLimit 付き |

### ウィジェット埋め込み仕様（LP 用）

```html
<script src="https://cdn.r2c.biz/widget.js"
  data-api-key="${LP_DEMO_API_KEY}"
  data-tenant="r2c_default"
  data-title="R2C デモ AI"
  data-color="#7c3aed"
  data-auto-open="true"
  data-greeting="こんにちは！R2Cのデモです。料金・機能・導入方法など何でも聞いてください。"
  defer>
</script>
```

### `r2c_default` システムプロンプト調整（LP 向け）

LP 訪問者（見込み顧客）に特化した応答に調整:
- R2C のサービス概要・料金・導入フローに特化して回答
- 「資料を送ってほしい」「詳しく話したい」 → Lead Form へ誘導するセリフを出力
- 技術的すぎる質問・競合比較は「担当からご説明します」と誘導

### 非機能要件

| 項目 | 要件 |
|---|---|
| Mobile First | 390px viewport ファースト / タッチターゲット 44px 以上 |
| パフォーマンス | LCP ≤ 2.5s（ウィジェットは defer ロード） |
| アクセシビリティ | WCAG 2.1 AA（color contrast / aria-label） |
| SEO | OGP / Twitter Card / meta description / JSON-LD |
| セキュリティ | CSRF（honeypot） / rateLimit / XSS（Shadow DOM 隔離） |
